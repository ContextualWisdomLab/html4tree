## 2026-06-24 - HTML XSS 취약점 및 Kotlin 템플릿 인젝션 방지
**Vulnerability:** `html4tree` 애플리케이션에서 파일 및 디렉토리 이름을 기반으로 `index.html`을 생성할 때 XSS 취약점이 존재했습니다. 악의적인 디렉토리/파일 이름(예: `<script>alert(1)</script>`)이 HTML 템플릿에 이스케이프되지 않고 직접 삽입되었습니다.
**Learning:** 파일 시스템 경로나 이름도 신뢰할 수 없는 입력으로 취급해야 하며, HTML로 렌더링될 때는 반드시 적절한 인코딩(HTML 엔티티 이스케이프 등)이 필요합니다. 또한 속성값에 들어가는 경우 반드시 따옴표(`"` 또는 `'`)로 감싸야 속성 탈출(Attribute Breakout) 공격을 막을 수 있습니다.
**Prevention:** Kotlin의 문자열 템플릿 내에 사용자나 파일 시스템으로부터 온 동적 값을 삽입할 경우 `.escapeHtml()`과 같은 확장 함수를 구현하여 `<, >, &, ", '` 등의 특수 문자를 필터링해야 합니다.
