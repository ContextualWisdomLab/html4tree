## 2024-05-24 - [CRITICAL] Fix Cross-Site Scripting (XSS) in Directory Index Generator
**Vulnerability:** The `html4tree` directory index generator was reading local directory names and injecting them directly into generated HTML elements (like `<title>`, `<h1>`, and list items) and `href` attributes without escaping HTML entities or URL encoding. This means that a malicious file or directory name like `<script>alert(1)</script>` would be executed as a script.
**Learning:** Even internal or local-only tools must sanitize file and directory names. Directory names are considered untrusted input when creating web artifacts. The application also missed basic attribute encapsulation (`href="..."`), making attribute injection very easy.
**Prevention:** Always implement an HTML escaping function (`&` -> `&amp;`, `<` -> `&lt;`, `>` -> `&gt;`, `"` -> `&quot;`, `'` -> `&#x27;`) for rendering user-controlled or system-read content. For link URLs, always URL encode the string to safely pass it in standard `href` attributes, and always encapsulate the attribute value in double quotes (`"`).
